# EverBee Store App Template

Build your EverBee Store app in minutes with AI assistance.

This template provides a complete full-stack foundation for building apps on the EverBee Store platform. Fork it, run setup, build with Cursor, deploy, and submit to the EverBee ecosystem.

## What's Included

- **Frontend**: React 18 (Vite) with TypeScript, Tailwind CSS
- **Backend**: Node.js (Express) with TypeScript
- **Authentication**: Complete OAuth flow with EverBee Store
- **Database**: MongoDB Atlas with Mongoose ODM
- **Pre-built Pages**: Dashboard, products, orders, analytics
- **EverBee SDK**: Pre-integrated API client
- **AI-Ready**: Optimized for Cursor / Claude Code
- **Demo Data**: Working app out of the box
- **One-Command Deploy**: Vercel (frontend) + Railway (backend)

## The Flow

```
Fork & Clone  -->  npm run setup  -->  Build with Cursor  -->  npm run deploy  -->  Submit to EverBee
```

## Quick Start

### Prerequisites

- **Node.js 18+** ([download](https://nodejs.org/))
- **MongoDB Atlas account** (free tier, [sign up](https://www.mongodb.com/atlas))
- **EverBee Store account** ([developer dashboard](https://store.everbee.io/admin/apps))

### 1. Fork and clone

```bash
# Fork this repo on GitHub, then:
git clone https://github.com/YOUR_USERNAME/store-app-template.git my-everbee-app
cd my-everbee-app
```

### 2. Run setup

```bash
npm run setup
```

The setup wizard will walk you through:
1. Naming your app
2. Connecting your MongoDB Atlas cluster
3. Adding your EverBee OAuth credentials
4. Installing dependencies and seeding demo data

### 3. Start developing

```bash
npm run dev
```

Open http://localhost:3000 -- you'll see a working app.

Demo login: `demo@example.com` / `password`

### 4. Build your features

Open the project in Cursor and start building:

```
"Add a chart showing sales by product"
"Create a bulk product upload feature"
"Add email notifications for new orders"
```

See [BUILD_WITH_AI.md](./BUILD_WITH_AI.md) for a detailed guide.

### 5. Deploy

```bash
npm run deploy
```

This deploys your frontend to Vercel and backend to Railway, wires up all environment variables, and prints your live URLs.

### 6. Submit to EverBee

Once deployed, submit your app to the EverBee Store ecosystem:

1. Go to https://store.everbee.io/admin/apps
2. Click **Create New App** (or **Submit App**)
3. Fill in:
   - **App Name**: your app name
   - **App URL**: your Vercel frontend URL
   - **OAuth Redirect URI**: `https://your-backend.up.railway.app/api/auth/callback`
4. Submit for review

## Environment Variables

All environment variables are collected during `npm run setup`. You can also edit them manually:

### Backend (`backend/.env`)

```bash
MONGODB_URI=mongodb+srv://...          # MongoDB Atlas connection string
EVERBEE_CLIENT_ID=your_client_id      # From EverBee developer dashboard
EVERBEE_CLIENT_SECRET=your_secret     # From EverBee developer dashboard
EVERBEE_REDIRECT_URI=http://localhost:3001/api/auth/callback
JWT_SECRET=auto-generated              # Auto-generated by setup
PORT=3001
FRONTEND_URL=http://localhost:3000
```

### Frontend (`frontend/.env`)

```bash
VITE_API_URL=http://localhost:3001
VITE_APP_NAME=My EverBee App
```

## Authentication Flow

The template implements EverBee's full OAuth flow:

1. User installs your app from the EverBee marketplace
2. Redirected to your app with an auth token
3. Your app exchanges the token for an access token
4. Access token is stored securely in MongoDB
5. All API calls use the stored token

Handled automatically in `backend/src/lib/everbee-auth.ts`.
